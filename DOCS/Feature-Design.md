# Feature Design

## 产品定位

ASMRPlayer 是面向 ASMR 音声收听场景的 Android 本地播放器，核心目标是管理本地音频、播放带字幕的音声、提供睡眠定时，并可选择接入 DLsite 购买库同步和下载。

## 本地资料库

- 支持创建、重命名、删除播放列表。
- 支持通过系统文件选择器导入单个或多个音频文件。
- 支持通过系统文件夹选择器导入目录，并自动扫描目录内音频。
- 导入文件夹时会尝试匹配字幕文件。
- 支持为播放列表设置封面。
- 支持曲目重命名、删除、移动到其它播放列表。
- 支持为单个曲目绑定字幕文件。
- 保存当前选中的播放列表。

## 播放

- 使用 Media3 `ExoPlayer` 播放音频。
- 使用 Media3 `MediaSessionService` 对接系统媒体会话。
- 支持播放、暂停、停止、上一首、下一首、进度跳转。
- 支持播放播放列表中的指定曲目。
- 支持播放队列展示当前列表和接下来播放的曲目；队列页不提供随机播放或拖拽排序入口。
- 播放状态会同步到 UI、Mini Player、播放页和睡眠模式。
- 不提供独立播放音质档位；本地导入、外部加载和下载音频均按源文件交由 Media3 播放。

## 字幕

- 支持绑定字幕文件并在播放页显示。
- 支持当前句、上一句、下一句的字幕式播放页展示。
- 支持中日双语字幕拆分显示。
- 支持桌面悬浮字幕。
- 悬浮字幕支持显示、隐藏、锁定和解锁。
- 悬浮字幕提供上一首、播放/暂停、下一首等控制入口。
- 支持为曲目自动生成 AI 字幕：默认在本机做日语转写，也可选择远程 ASR 转写服务；先生成日文单语中间 `.vtt`，翻译完成后生成仅含中文译文的 `.vtt` 并自动绑定。
- AI 字幕生成文件名包含曲目 ID；外部导入的多条音频即使标题清洗后相同，也会写入不同 `.vtt`，避免后生成的字幕覆盖其它曲目已经绑定的字幕文件。
- AI 字幕转写使用 sherpa-onnx Whisper + Silero VAD CPU 后端：音频解码为 16k mono PCM 后由 VAD 流式切成语音片段，片段在送入 Whisper 前会按默认 0.6s 邻近阈值合并相邻短段，多个 Whisper worker 边解码边消费识别。
- VAD 短段合并不插入额外静音，合并后音频样本只包含真实语音；字幕时间轴使用首段 `startMs` 与末段真实 `endMs`，避免句中停顿把一句话拆成多条缺主干短行。
- CPU Whisper 模型按需下载到应用私有目录，`base` 为默认模型，`small` 为可选模型；模型文件包含 encoder、decoder、tokens 和 Silero VAD。
- 可选远程 ASR 后端会把整段音频上传到用户配置的服务器 `POST /transcriptions` 创建异步转写任务，服务端返回 `job_id` 后由 App 轮询 `GET /transcriptions/{job_id}` 获取 `status`、`stage`、`progress`、`message` 和错误信息；完成后调用 `GET /transcriptions/{job_id}/result` 取得 `segments[{id,start_ms,end_ms,text}]` 并映射为现有 `SubtitleLine`，取消时调用 `DELETE /transcriptions/{job_id}`。
- 远程 ASR 服务基线模型组合为 `Qwen3-ASR-0.6B` + `Qwen3-ForcedAligner-0.6B`；ASR 负责日语识别，ForcedAligner 负责把文本对齐回音频时间轴，输出完整句粒度字幕段。服务可支持其它模型，但必须遵守 `DOCS/tech/remote-asr-progress-contract.md` 的字段、错误码和进度更新要求。
- 远程 ASR 后端支持分开配置服务器地址、端口、可选转写模型和可选 Bearer Token；设置页「测试连接」调用 `GET /health` 检查服务可达、鉴权可用和 `models_ready`，可用时只显示 `OK`，失败、鉴权错误或模型未就绪时显示具体原因；排队满、对齐失败等任务级错误在生成字幕流程中显示。
- 转写进度由解码/VAD 阶段和识别阶段组成，解码进度驱动 VAD 权重，已识别片段时长驱动识别权重，并保持 UI 进度不回退。
- 远程转写进度由服务端 job 状态驱动：上传阶段显示本地上传进度；提交成功后 UI 使用服务端返回的 `progress` 与 `stage`，要求单调不回退，并覆盖排队、ASR、强制对齐、写出结果、失败和取消状态。
- Whisper 识别并发会按模型体量、可用内存、低内存状态、CPU 核数和片段数规划；`base` 在多核充裕内存设备可使用更多 worker，`small` 或低内存设备会降低 worker 数以避免并发加载多份模型导致 OOM。
- AI 字幕转写完成后会把分段时间轴与日文原文暂存到应用缓存目录；缓存键区分本机/远程后端、本机模型或远程地址/端口/model，翻译失败后点重试会优先复用匹配缓存，避免重新跑本机转写或重新提交远程 job。
- AI 字幕翻译前会生成或复用全局情景卡，包含场景、说话者、听者称呼、语气和术语表；情景卡失败时降级为空情景，不阻断翻译。
- AI 字幕翻译会按约 80 行固定批次保存已完成译文；每批附带只读前 4 行“日文 + 已定中文”和后 4 行日文窗口，用于保持称呼、语气和术语一致。
- AI 字幕翻译使用 OpenAI 兼容 `chat/completions`，内置本地 Ollama 与云端 OpenAI 兼容接口两种模式；云端模式默认预设为 DeepSeek `https://api.deepseek.com` + `deepseek-v4-flash`，但接口地址、模型名和 API Key 都可自定义，任意非空模型名会原样保存，只有留空时才回退默认模型。
- 云端 OpenAI 兼容翻译会把完整日文字幕全文作为只读全局上下文放进每批请求的稳定前缀，帮助模型先理解全文背景；输出目标仍只限当前批次，Ollama 不携带全文上下文以避免本地小上下文模型失稳。
- AI 字幕翻译请求使用较长的 LLM 超时设置，并把超时、域名解析和 HTTP 错误转为中文提示。
- AI 字幕翻译请求/响应使用有序 `lines` 数组协议：输入只含 `{id, ja}`，输出必须是同长度、同顺序、同 id 的 `{id, zh}`；prompt 不包含任何时间信息，最终 VTT 时间轴完全沿用源 `SubtitleLine.startMs/endMs`。
- AI 字幕翻译采用逐行保守策略：上下文、标题、情景卡和术语表只用于消歧，不能替代当前行；短碎片、半个词、疑似 ASR 误识别或无法确定含义时，会输出中文保守占位，不再保留日文，避免凭空补人物、感受或动作。
- AI 字幕翻译会把 ASMR 非台词声音按音效处理：舔舐、吸吮、摩擦、衣料/耳边摩擦、呼吸、喘息或湿润口腔音等拟声不硬译字面，优先输出「（舔舐声）」「（摩擦声）」「（呼吸声）」这类简短中文声音描述。
- AI 字幕翻译响应会校验合法 JSON、`lines` 数组、行数、id 顺序、id 唯一性、非空译文和平假名残留；叹息、喘息、拟声、符号或无实义短句也要求输出中文拟声、中文音效描述或中文保守占位。若完成结果中仍含片假名，不阻断字幕保存和绑定，而是在任务完成态、生成 sheet 和通知中提示需要人工检查。
- AI 字幕设置提供“成人内容直译”开关；开启后翻译 prompt 会要求按原文直接翻译成人向敏感词，不用省略或模糊说法规避，但仍不得添加当前日文原文没有的成人内容。
- AI 字幕翻译空 content、`finish_reason=length` 截断、缺句、乱序、重复 id 或空译会整批自动重试；整批连续失败后降级逐句翻译，保证缺一行就失败而不是生成可能错位的 VTT。
- AI 字幕翻译缓存签名包含协议版本、情景卡版本、批策略版本、情景卡签名、模型、接口地址和源字幕签名；重试时会跳过已完整翻译的批次，只重新请求缺失或失败的批次。
- AI 字幕生成以后台前台服务运行，提供转写/翻译两阶段进度、实时片段预览、暂停、继续、失败重试和取消入口；已完成或失败的生成任务可在当前生成字幕 sheet 中点“重新分片并翻译”，强制清除分段、情景卡和翻译缓存后按当前后端重新转写与翻译。
- 普通失败重试会继续利用已匹配的分段/翻译缓存，适合网络或模型响应失败；“重新分片并翻译”用于分句切得太碎、源字幕识别质量差或需要完全重做的场景。

## 睡眠定时

- 支持按分钟设置睡眠倒计时，点击预设分钟按钮会立即设置并启动对应分钟倒计时。
- 支持播完当前音频后停止。
- 支持自定义睡眠时间；底部单一入口在未计时时打开自定义时间，计时启动后切换为取消定时。
- UI 会在圆环中心显示剩余倒计时或播完当前音频模式图标，不在圆环下方显示额外状态文案。

## DLsite

- 支持通过 WebView 登录 DLsite。
- 支持检测 DLsite 登录 Cookie。
- 支持同步已购买作品。
- 支持下载作品封面。
- 支持请求 DLsite Play 文件目录树，并按目录/根目录音频生成可缓存下载内容清单。
- DLsite 文件目录树解析产物使用 `domain.model.DlsiteZiptree` 与 `DlsiteContentFile`；下载规划和下载选项不依赖 remote parser 内部 DTO。
- 支持下载作品文件到应用私有目录。
- 支持下载队列，多个作品按加入顺序排队，并发上限固定为 2。
- 下载队列持久化在 `dlsite_download_queue`；服务重启后会把遗留 `running` 任务重置为 `pending`，再按 FIFO 调度。
- 同一作品已有 active `pending` / `running` 任务时再次入队会去重；异常重复 `pending` 不阻塞后续作品调度。
- 点击加入下载只入队并启动下载服务，排队或开始下载不会更新 `DlsiteWork` / `DlsiteContent` 的 `updatedAt`，作品列表不会因此立即置顶。
- 支持 DLsite 页顶部显示总下载进度、总速度、已下载/总大小，以及全部暂停/继续。
- 支持下载管理 sheet 统一查看任务，并对单任务暂停、继续、取消，或清除已完成任务。
- 支持 DLsite 作品行保持解析/展开入口；内容级下载不会把作品行改成整体排队、整体载入、整体已载入或部分载入状态。
- 支持在 DLsite 作品行通过同步按钮解析并缓存可下载内容，通过内容行按钮单独下载或重试某个内容。
- 支持内容级增量下载：作品下载内容以 `DlsiteDownloadOption` 为粒度保存状态，当前粒度来自 DLsite 文件目录树中的目录/根目录音频；已下载内容会跳过，新增内容追加到已有播放列表，不再把内容状态回写成作品整体已下载状态。
- 单个 DLsite 下载任务会复用同一个播放列表，避免同一内容被导入成两个同名列表。
- 支持在作品卡内以 accordion 方式展开或合上已解析并缓存的文件目录树选项；内容级下载中和下载完成后仍保持该目录展示逻辑，不把作品卡切换成独立下载中或整体完成操作区；再次同步同一作品时优先复用本地内容清单，不重复请求 DLsite 目录树。
- 作品卡内的内容清单以逐项操作为主，下载状态由左侧勾选图标呈现；已下载或检测到本地缓存的内容会点亮勾选图标并把右侧操作切换为删除，其它内容项右侧提供下载、重试、暂停或继续按钮；内容级下载任务会保持作品卡展开，并在对应内容项下方显示进度线，方便继续下载其它部分。
- 单次选择的所有内容都下载成功后才统一导入或复用本地播放列表，再按内容回填 `trackIds`；单个内容包完成时不会提前导入，避免资料库出现半成品。
- 支持保存 DLsite 作品列表、内容级下载状态、封面路径和上次同步时间。

## 设置

- 支持浅色、深色、跟随系统主题模式。
- 主题模式会保存到 Room 应用设置的 `app_theme_mode`，重启后由 `SettingsRepository.themeModeFlow` 恢复到设置页和应用主题。
- 支持悬浮字幕开关。
- 支持悬浮窗权限入口。
- 支持 Android 13 及以上通知权限请求。
- 支持在设置页高级设置中进入「AI 字幕与翻译」二级页面，配置 AI 翻译接口、本地 Ollama / 云端 OpenAI 兼容接口地址、模型、API Key、成人内容直译开关、转写后端（本机 / 远程服务器）、Whisper base/small 模型下载管理，以及远程 ASR 服务地址、端口、可选转写模型、Bearer Token、连接诊断和异步任务状态。
- 支持在设置页显示当前软件版本号。
- 支持手动检查 GitHub Release 最新版本，使用 `tag_name` 与本地 `versionName` 比较。
- 检查更新支持检查中、已是最新、发现新版本和检查失败状态。
- 发现新版本后显示更新详情弹窗，展示版本号、更新日志和立即更新入口。
- 支持以前台服务下载 Release asset 中的 APK，切到后台后仍通过“应用更新”通知显示百分比、速度、已下载/总大小和取消操作。
- 设置页更新下载卡会同步显示细线进度条、速度、已下载/总大小、取消和失败重试。
- 更新包下载完成后弹出安装确认；Android 8 及以上会处理“安装未知应用”授权后拉起系统安装器。
- 更新 APK 缓存位于 `cacheDir/updates/`；下载前清理旧版本 APK 和遗留 `.part`，下载失败或取消会删除当前 `.part`。安装成功收到应用替换广播后会删除当前已安装版本及更旧的 `ASMRPlayer-*.apk` 和全部 `.part`，应用启动时也会保守清理，但不会删除比当前安装版本更新的 APK。

## 数据与持久化

- Room 数据库存储播放列表、曲目、DLsite 作品、DLsite 内容下载状态、DLsite 持久化下载队列和应用设置。
- 当前数据库版本为 6。
- 已配置 v1 到 v6 的 Room migration。
- 使用 Repository 层隔离 UI/ViewModel 与数据库/API。
- Repository 以 `Flow` 推送播放列表、DLsite 内容、上次同步时间和 AI 设置等状态；写入和必要快照读取在 IO 协程中执行，UI 通过 ViewModel state/event 接收更新，不直接同步阻塞读取 Room。
- `DlsiteRepository` 只暴露 DLsite 作品、内容、远程同步和下载状态协调能力；持久下载队列状态机由 `DlsiteDownloadQueueRepository` 独立管理并显式注入调用方。
- AI 字幕设置复用应用设置表；Whisper 模型和生成的 `.vtt` 存放在应用私有目录 `filesDir/ai-subtitles/`，转写分段缓存存放在 `cacheDir/ai-subtitles/segments/`，情景卡缓存存放在 `cacheDir/ai-subtitles/scene-contexts/`，翻译批次缓存存放在 `cacheDir/ai-subtitles/translations/`。远程转写配置也保存在设置表，不触发 Room schema migration。

## 内部验证能力

- 提供 `uiProbe` 内部构建变体，独立包名为 `io.github.summerdez.asmrplayer.uiprobe`，不会覆盖正式包。
- `uiProbe` 构建显示右上角“UI 探针”控件；激活后点选界面，会高亮命中的最小 UI 区域，并在底部面板显示 `selected.id`、中文 label、来源文件和坐标。
- 点选结果会写入应用私有目录 `files/ui-probe/latest-selection.json`，并通过 `ASRM_UI_PROBE` Logcat tag 输出，供 AI 或开发者把用户标记的视觉区域映射到 Compose 源文件和业务对象。
- `uiProbe` 仅用于 UI 定位、截图和验证；发布、上架和正式回归仍使用 release APK。
